# Worklog 2026-06-28

## Forward CLV calibration

- Added `node src/cli.mjs clv-calibrate`.
- The command is offline-only: it reads `reports/paper-bets.csv`, excludes rows
  without valid numeric `ev` and captured `clv`, and writes:
  - `reports/clv-calibration.csv`
  - `reports/clv-calibration.json`
- Report dimensions:
  - overall
  - EV bucket (`<-5%`, `-5..0%`, `0..2%`, `2..5%`, `5..10%`, `10%+`)
  - tier (`VALUE`, `CONTROL`, `VALUE_CHECK`, etc.)
  - sportKey
  - market
  - bookmaker
  - odds bucket
- Report metrics:
  - count
  - positive CLV rate
  - average EV
  - average CLV
  - median CLV
  - CLV minus EV
  - global `clv ~ computed_ev` linear regression slope/intercept/R^2

## Soccer-only event-market expansion

Goal:

- Increase forward VALUE CLV volume without adding US-style markets.
- Keep `MATCH_RESULT` as the main benchmark.
- Keep every market in separate calibration buckets.

Implemented:

- Added `theodds-sweep --market-profile=soccer-core`.
- The profile filters to `soccer_*` sports only.
- It uses The Odds API Event Odds endpoint for:
  - `h2h`
  - `h2h_3_way`
  - `draw_no_bet`
  - `btts`
  - `double_chance`
- It does not add spreads, team totals, alternate lines, player props, or other
  US-style markets.
- Added safety knobs:
  - `--event-limit=N`
  - `--max-event-credits=N`
- Added normalization:
  - `draw_no_bet` -> `DRAW_NO_BET`
  - `h2h_3_way` -> existing `MATCH_RESULT`
- Extended paper CLV capture so pending soccer `DRAW_NO_BET`, `BTTS`, and
  `DOUBLE_CHANCE` rows call event odds near close. This prevents new research
  rows from becoming uncapturable paper-only rows.

First capped live run:

```powershell
node src/cli.mjs theodds-sweep --market-profile=soccer-core --edge=0 --sample-min-ev=-5 --sample-limit=150 --max-sports=8 --event-limit=4 --max-event-credits=150
```
