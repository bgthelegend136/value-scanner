## 1. Τι κάνει αυτό το εργαλείο

Βρίσκει **value bets** (στοιχήματα με θετική αξία) σε **όλες τις λίγκες που είναι
σε σεζόν** και υπάρχουν στο `config/multisport-map.json` (Μουντιάλ, Brazil Série
A/B, EPL, Serie A, League of Ireland, Superettan, MLB, NPB, NFL, …):

1. Τραβάει τις αποδόσεις που δίνουν **Stoiximan** και **Superbet** (εκεί ποντάρεις).
2. Τις συγκρίνει με τη «δίκαιη» τιμή που προκύπτει από την **Pinnacle** (τη σοβαρότερη
   στοιχηματική) και από τον **μέσο όρο όλης της αγοράς**.
3. Όπου μια ελληνική στοιχηματική σου δίνει **καλύτερη απόδοση από τη δίκαιη**,
   το σημειώνει ως ευκαιρία.

**Δεν** κάνει scraping, **δεν** ποντάρει μόνο του, **δεν** στέλνει ειδοποιήσεις σε άλλους.
Είναι εργαλείο **προσωπικής ανάλυσης**.

## 3. Πώς το τρέχεις

Άνοιξε PowerShell στον φάκελο:

```powershell
cd C:\Users\bgthe\Documents\bet\provider-harness
```

### Η κύρια εντολή: `scan`
```powershell
node src/cli.mjs scan
```
Σαρώνει **όλες τις σε-σεζόν λίγκες** (όχι μόνο Μουντιάλ), βρίσκει τα value bets,
τα τυπώνει, γράφει CSV στο `reports\` και τα καταγράφει ως **paper bets**. Το
προεπιλεγμένο κατώφλι είναι πλέον **2%** (το ρεαλιστικό sharp edge). Δεν στέλνει
τίποτα σε Telegram — είναι μόνο για συλλογή μετρήσιμων δεδομένων (paper + CLV).

Πιο αυστηρό κατώφλι (λιγότερα, καθαρότερα):
```powershell
node src/cli.mjs scan --edge=5
```

### Έλεγχος ότι όλα δουλεύουν (δωρεάν, χωρίς κλειδιά)
```powershell
node --test
```
Πρέπει να δεις `# pass 61  # fail 0`.

### Παρακολούθηση απόδοσης (paper ROI)
Κάθε `scan` καταγράφει **αυτόματα** τα value bets σε ένα μόνιμο ημερολόγιο
`reports/paper-bets.csv` (paper bet, 1 μονάδα stake, χωρίς διπλά). Δες παρακάτω
τη **Καθημερινή ροή** για πότε ακριβώς να τρέχεις τι.

Όταν τελειώσουν οι αγώνες, τρέξε:
```powershell
node src/cli.mjs fd-settle   # ΔΩΡΕΑΝ: settle soccer/Μουντιάλ μέσω football-data.org
node src/cli.mjs settle      # The Odds API: μόνο για τα non-soccer που μένουν
```
Το **`fd-settle`** κλείνει τα soccer στοιχήματα **δωρεάν** (football-data.org,
χωρίς The Odds API credits) — τρέξ' το **πρώτα**. Μετά το `settle` τραβάει σκορ
μόνο για ό,τι δεν καλύπτει το football-data (π.χ. baseball/NFL). Και τα δύο βρίσκουν
WON/LOST/PUSH και τυπώνουν **πραγματικό ROI**. Έτσι βλέπεις αν τα +EV σήματα όντως
αποδίδουν — μέσα στις μέρες του Μουντιάλ.

⚠️ Είναι **paper** (χάρτινο) μέτρημα, όχι απόδειξη μελλοντικού κέρδους. Στα νοκ-άουτ,
το σκορ μπορεί να περιλαμβάνει παράταση — προσοχή σε τέτοιες περιπτώσεις.

### CLV — Closing Line Value
Λίγο **πριν** την έναρξη κάθε αγώνα, τρέξε:
```powershell
node src/cli.mjs clv
```
Ξαναβλέπει τη γραμμή της Pinnacle τελευταία στιγμή και μετράει αν η απόδοση του bet σου
**νίκησε τη γραμμή κλεισίματος** (θετικό `clv`). Είναι το πιο **γρήγορο** σήμα ότι το edge
ήταν αληθινό — χωρίς να περιμένεις αποτέλεσμα.

Τυπώνει κάτι σαν:
```
CLV captured: 3 | Beat rate: 100.0% | Avg CLV: +3.5%
```
Αν το `Beat rate` μείνει ψηλά μακροπρόθεσμα → το σύστημα βρίσκει πραγματικό edge.

### Άλλες εντολές (προαιρετικές)
- `node src/cli.mjs events` — λίστα αγώνων.
- `node src/cli.mjs capture <eventId>` — φύλλο για χειροκίνητη σύγκριση τιμών.
- `node src/cli.mjs evaluate <capture.csv>` — αξιολόγηση χειροκίνητων παρατηρήσεων.

## 4. Πώς διαβάζεις τα αποτελέσματα

Κάθε scan γράφει **δύο** αρχεία στο `reports\`:
- `scan-<ημερομηνία>.csv` → **μόνο οι ευκαιρίες**, ταξινομημένες κατά EV. **Αυτό κοιτάς.**
- `scan-all-<ημερομηνία>.csv` → όλα όσα εξετάστηκαν (για έλεγχο/audit).

Άνοιξε το `scan-<...>.csv` με διπλό κλικ (Excel/Sheets).

### Οι στήλες
| Στήλη | Τι σημαίνει |
|---|---|
| `ev` | Το πλεονέκτημα, π.χ. `+4.7%`. Όσο μεγαλύτερο, τόσο καλύτερα. |
| `tier` | Κατηγορία εμπιστοσύνης (δες παρακάτω). |
| `match` | Ο αγώνας. |
| `pick` | Η επιλογή: `Home (1)`, `Draw (X)`, `Away (2)`, ή `OVER/UNDER 2.5`. |
| `bookmaker` | Πού ποντάρεις: `Stoiximan` ή `Superbet`. |
| `odd` | Η απόδοση που σου δίνει η στοιχηματική. |
| `fairOdd` | Η «δίκαιη» τιμή κατά **Pinnacle** (ο βασικός χάρακας). |
| `marketFair` | Η «δίκαιη» τιμή κατά **όλη την αγορά** (~20+ books). |
| `books` | Πόσες στοιχηματικές στηρίζουν το `marketFair`. |
| `kickoffUtc` | Ώρα έναρξης (UTC). |

### Οι κατηγορίες (tier)
- **`VALUE`** (EV 3–5%) → η πιο **αξιόπιστη** ζώνη. Εδώ ζει το πραγματικό edge.
- **`VALUE_CHECK`** (EV 5–15%) → ευκαιρία, αλλά **τσέκαρε** γραμμή/ώρα/σύνθεση πρώτα.
- **`SUSPICIOUS`** (EV >15%) → σχεδόν πάντα **λάθος/παλιά γραμμή**, όχι «δυνατότερο»
  στοίχημα. Εμφανίζεται με προειδοποίηση.

### Ο κανόνας
- **`odd > fairOdd`** → value έναντι Pinnacle.
- Αν επιπλέον **`odd > marketFair`** → το επιβεβαιώνει κι όλη η αγορά (πιο δυνατό σήμα).
- Αν **`odd < marketFair`** ενώ `> fairOdd` → value μόνο κατά Pinnacle· η αγορά διαφωνεί
  → πιο επιφυλακτικά.

## 5. Οι έννοιες (απλά)

### Δίκαιη τιμή (fair odd)
Η απόδοση που αντιστοιχεί στην **πραγματική πιθανότητα**, χωρίς το κέρδος της εταιρείας.
Παράδειγμα: αν η δίκαιη πιθανότητα για ισοπαλία είναι 50%, η δίκαιη απόδοση είναι
`1 / 0,50 = 2,00`.

### De-vig (αφαίρεση περιθωρίου)
Κάθε στοιχηματική φουσκώνει τις τιμές για να βγάλει κέρδος — γι' αυτό οι πιθανότητες
1/Χ/2 αθροίζουν πάνω από 100%. Παίρνουμε τις τιμές της Pinnacle, «ξεφουσκώνουμε» πίσω
στο 100%, και μένει η καθαρή/δίκαιη πιθανότητα.

Χρησιμοποιούμε τη μέθοδο **power** (αντί της απλής αναλογικής), που διορθώνει το
*favorite-longshot bias*: τα μεγάλα underdogs τείνουν να φαίνονται ψευδώς «value».
Έτσι φουσκωμένα longshots (π.χ. απόδοση 28 σε αουτσάιντερ) **δεν** περνάνε λάθος ως
ευκαιρίες — το EV γίνεται πιο ρεαλιστικό.

### Γιατί Pinnacle
- Παίρνει **ελάχιστο** περιθώριο (~2-3%).
- **Δεν** μπλοκάρει τους νικητές — δέχεται επαγγελματίες, άρα οι τιμές της «διορθώνονται»
  από το έξυπνο χρήμα. Είναι σαν **τιμή μετοχής**: η καλύτερη διαθέσιμη εκτίμηση.

### EV (Expected Value)
`EV = (απόδοση × δίκαιη_πιθανότητα) − 1`. Θετικό EV = μακροπρόθεσμα κερδοφόρο **κατά μέσο
όρο**, αν η εκτίμηση είναι σωστή.

### Consensus (μέσος όρος αγοράς)
Το ίδιο de-vig, αλλά μέσος όρος **πολλών** books — context για να δεις αν η Pinnacle
συμφωνεί με την υπόλοιπη αγορά.

Το ημερολόγιο ανοίγει με Excel/Sheets. Κάθε γραμμή = ένα bet.

| Στήλη | Τι σημαίνει |
|---|---|
| `referenceEventId` | Αναγνωριστικό αγώνα στο The Odds API (Pinnacle) |
| `bettableEventId` | Αναγνωριστικό αγώνα στο Odds-API.io (Stoiximan/Superbet) |
| `firstSeenAt` | Πότε εντοπίστηκε η ευκαιρία (UTC) |
| `kickoffUtc` | Έναρξη αγώνα (UTC) |

| `bookmaker` | Πού ποντάρεις: `Stoiximan` ή `Superbet` |
| `market` | Αγορά: `MATCH_RESULT` (1Χ2) ή `TOTALS` (πάνω/κάτω) |
| `line` | Γραμμή για TOTALS (π.χ. `2.5`), κενό για 1Χ2 |
| `outcome` | Επιλογή: `1`, `X`, `2`, `OVER`, `UNDER` |
| `decimalOdds` | Απόδοση της στοιχηματικής όταν εντοπίστηκε |
| `fairOdds` | Δίκαιη τιμή Pinnacle (de-vig) |
| `fairProbability` | Δίκαιη πιθανότητα (π.χ. `0.158` = 15.8%) |
| `ev` | Expected Value: `+0.038` = +3.8% πλεονέκτημα |
| `tier` | `VALUE` / `VALUE_CHECK` / `SUSPICIOUS` |
| `stake` | Μονάδες stake (πάντα `1.00` στο paper tracking) |
| `status` | `PENDING` → `WON` / `LOST` / `PUSH` / `REVIEW` |
| `homeScore` / `awayScore` | Τελικό σκορ (μετά το `settle`) |
| `profit` | Κέρδος σε μονάδες (π.χ. `5.5600` αν κέρδισες, `-1.0000` αν έχασες) |
| `settledAt` | Πότε εκδόθηκε αποτέλεσμα |
| `closingFairOdds` | Δίκαιη τιμή Pinnacle λίγο πριν την έναρξη (από `clv`) |
| `clv` | Closing Line Value: `+0.038` = νίκησες τη γραμμή κατά 3.8% |
| `clvCapturedAt` | Πότε καταγράφηκε το CLV |

## 8. Ασφάλεια & όρια

- **Κανένα scraping/automation** σε ιστοσελίδες στοιχηματικών. Όλα μέσω επίσημων API.
- **Καμία αυτόματη πονταρίσματα** — εσύ αποφασίζεις και ποντάρεις χειροκίνητα.
- Τα κλειδιά μένουν στο `.env.local`, **ποτέ** σε logs/reports/git.
- Οι raw απαντήσεις των API **δεν** αποθηκεύονται — μόνο καθαρά δεδομένα σύγκρισης.
- **Ταυτότητα feed:** το odds-api.io δήλωσε ότι **δεν** δίνει απομονωμένο ελληνικό feed,
  οπότε το «Stoiximan» του είναι ένα ευρωπαϊκό feed με ετικέτα Stoiximan — **όχι**
  εγγυημένα οι τιμές του stoiximan.gr. Το εργαλείο το αναφέρει ως `UNVERIFIED`.
- **Στοίχημα = ρίσκο.** Το θετικό EV είναι στατιστικό πλεονέκτημα, **όχι** σίγουρο κέρδος.
  Μεμονωμένα στοιχήματα χάνουν συχνά. Ποντάρισε υπεύθυνα.

## 11. Telegram ειδοποιήσεις για λάθη αποδόσεων

Το `mispricing-scan` είναι ξεχωριστό mode για μεγάλες αποκλίσεις σε Stoiximan
ή Superbet. Στην πρώτη έκδοση ελέγχει μόνο pre-match νικητή αγώνα /
moneyline / 1Χ2, αλλά όχι μόνο ποδόσφαιρο ή Μουντιάλ: εξετάζει candidates από
οποιοδήποτε sport επιστρέφει ο provider.

Ένα Telegram alert στέλνεται μόνο όταν:

- το αρχικό candidate έχει τουλάχιστον 10% EV,
- η δική μας επανεκτίμηση είναι **αυστηρά πάνω από 10%** έναντι de-vigged
  Pinnacle,
- είναι επίσης αυστηρά πάνω από 10% έναντι median consensus τουλάχιστον τριών
  άλλων διεθνών bookmakers,
- ταιριάζουν ακριβώς αγώνας, επιλογή και αγορά,
- τα δεδομένα είναι φρέσκα και ο αγώνας δεν έχει αρχίσει.

Η λίγκα πρέπει να υπάρχει και στον reference provider. Οι μοναδικές ακριβείς
αντιστοιχίες ενεργών τίτλων γίνονται αυτόματα, ενώ γνωστά aliases βρίσκονται
στο `config/multisport-map.json`. Αν δεν υπάρχει reference coverage ή η
αντιστοίχιση είναι αμφίβολη, δεν στέλνεται alert.

Το 10,0% ακριβώς στην τελική επιβεβαίωση δεν περνά.

### Ρύθμιση Telegram

Στο root `.env.local` πρόσθεσε:

```text
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...
```

Το token δημιουργείται από το επίσημο `@BotFather`. Στείλε πρώτα ένα μήνυμα
στο bot και βρες το προσωπικό chat id μέσω της επίσημης Telegram Bot API
μεθόδου `getUpdates`. Μην βάζεις token σε screenshots, logs ή commits.

Δοκιμή σύνδεσης:

```powershell
node src/cli.mjs telegram-test
```

Δοκιμή χωρίς αποστολή Telegram:

```powershell
node src/cli.mjs mispricing-scan --dry-run
```

Κανονική εκτέλεση:

```powershell
node src/cli.mjs mispricing-scan
```

Το κουμπί του Telegram χρησιμοποιεί μόνο allowlisted HTTPS link. Προσπαθεί να
ανοίξει πρώτα την ακριβή επιλογή και διαφορετικά τη σελίδα αγώνα. Πάντα έλεγξε
χειροκίνητα bookmaker, περιοχή, αγώνα, αγορά, επιλογή και τρέχουσα απόδοση.
Ειδικά τα Superbet links του provider μπορεί να δείχνουν στο
`superbet.bet.br` αντί για το ελληνικό προϊόν.

### Αυτόματη εκτέλεση Windows

Αφού περάσουν το `telegram-test` και ένα `--dry-run`, εγκατάστησε το task:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\install-mispricing-task.ps1
```

Τρέχει καθημερινά στις **09:00, 15:00 και 21:00**, με τοπική ώρα Windows.
Ο υπολογιστής πρέπει να είναι ανοιχτός, συνδεδεμένος στον λογαριασμό Windows
και online.

Έλεγχος:

```powershell
Get-ScheduledTask -TaskName Bet-Mispricing-Scanner
```

Προσωρινή απενεργοποίηση και επανενεργοποίηση:

```powershell
Disable-ScheduledTask -TaskName Bet-Mispricing-Scanner
Enable-ScheduledTask -TaskName Bet-Mispricing-Scanner
```

Η εγκατάσταση απενεργοποιεί το παλιό read-only task
`Bet-Mispricing-Funnel`, ώστε να μην καταναλώνουν και τα δύο API quota.
